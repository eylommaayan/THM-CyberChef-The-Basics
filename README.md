# 🛠️ CyberChef: המדריך המעשי ל-Penetration Tester

בעולם ה-PT, אנחנו נתקלים בנתונים "מעורפלים" (Obfuscated) כל הזמן: מחרוזות בתוך קוד JavaScript, עוגיות (Cookies) מקודדות, או תעבורת רשת חשודה. **CyberChef** הוא הכלי האולטימטיבי לפיצוח המבנים האלו במהירות.

## 🧱 מבנה העבודה (The Workflow)

בתור בודקי חוסן, זרימת העבודה שלכם תהיה תמיד:
1.  **Input:** הדבקת ה-Payload החשוד (Hash, Base64, Hex).
2.  **Search:** איתור הפעולה המתאימה ב-Operations (למשל: "Decode", "Decompress").
3.  **Recipe:** בניית "שרשרת פיצוח" (למשל: Base64 -> XOR -> Gunzip).
4.  **Output:** קבלת הקוד הקריא או ה-Flag.

## 🧭 ניווט מבצעי בממשק

### 🔍 Operations (ארגז הכלים)
אל תבזבזו זמן בחיפוש ידני בקטגוריות. השתמשו ב-**Search**:
* מחפשים משהו שקשור לכתובות אינטרנט? הקלידו `URL`.
* מחפשים להפוך קוד למובן? הקלידו `Beautify`.

### 👨‍🍳 Recipe (פס הייצור)
זה המקום שבו אתם בונים לוגיקה. 
* **סדר הפעולות הוא קריטי:** פתיחת קידוד (Decoding) חייבת לקרות לפני פענוח (Decryption).
* **ביטול זמני:** ניתן ללחוץ על האייקון של ה-"עצור" (Disable) ליד כל פעולה במתכון כדי לראות איך הפלט נראה בלעדיה, בלי למחוק אותה.

### 🪄 ה-Magic Wand (מטה הקסמים)
החבר הכי טוב שלכם ב-CTF. אם CyberChef מזהה תבנית מוכרת (כמו Base32 או קידוד Hex), יופיע מטה קסמים קטן ב-Output. לחיצה עליו תבנה לכם את המתכון אוטומטית.

## 🛠️ ארגז כלים מהיר ל-PT (Operations לשימוש יומיומי)

| כלי | שימוש ב-PT | סימן זיהוי |
| :--- | :--- | :--- |
| **From Base64** | חשיפת Payload ב-Web-Apps או Scripts. | תווים אקראיים, מסתיים לעיתים ב-`=`. |
| **URL Decode** | פיענוח פרמטרים בכתובת ה-URL (כמו `admin%3Dtrue`). | הופעת הרבה סימני `%`. |
| **Extract IP / URLs** | הוצאת רשימת כתובות מתוך קובץ לוג מבולגן. | שימוש ב-Regex מובנה לסינון נתונים. |

| **XOR** | פיצוח הצפנות פשוטות של Malware או Scripts. | דורש "Key" (מפתח) כדי לפענח. |

## 💡 טיפים למתקדמים (Pro-Tips)

1.  **עבודה עם קבצים:** אל תעתיקו טקסט מקבצים גדולים. גררו את הקובץ ישירות ל-**Input**. CyberChef יודע להתמודד עם קבצים בצורה מעולה.
2.  **Auto Bake:** בבניית מתכונים מורכבים (כמו XOR Brute Force), כבו את ה-Auto Bake כדי שהדפדפן לא יקפא, והריצו ידנית עם כפתור **BAKE!**.
3.  **Save Recipe:** בניתם מתכון מטורף שמפענח לוגים של שרת ספציפי? שמרו אותו (Save) לשימוש עתידי או שתפו את הקישור עם הצוות שלכם (המתכון נשמר בתוך ה-URL!).

---

**משימה הבאה:** נסה לקחת מחרוזת ב-Base64, להפוך אותה ל-Hex, ואז להפוך את הסדר (Reverse). זה ירגיל אותך לעבודה עם "מתכונים" מרובי שלבים.


👨‍🍳 אזור המתכון (Recipe Area) – "הלב המבצעי"
זהו האזור שבו בודק החוסן בונה את שרשרת העיבוד שלו. כאן אתה מגדיר את ה-"איך" – אילו פעולות יתבצעו ובאיזה סדר.

לוגיקה של שרשרת: ניתן לגרור פעולות ולשנות את סדרן. ב-PT זה קריטי; למשל, אם תנסה לפענח הצפנה לפני שביצעת From Base64, המתכון ייכשל.

ארגומנטים (Arguments): לכל פעולה יש הגדרות (כמו מפתח הצפנה או סוג תו מפריד). כאן אתה עושה Fine-tuning לכל כלי.

ניהול מתכונים:

Save/Load: מאפשר לשמור שרשראות מורכבות לשימוש חוזר (למשל, מתכון קבוע לפירוק לוגים של שרת מסוים).

BAKE!: הכפתור שמריץ את התהליך.

Auto Bake: חובה לעבוד איתו ברוב הזמן כדי לראות שינויים בזמן אמת, אך מומלץ לכבות אותו כשעובדים עם כמויות ענק של דאטה כדי למנוע קריסה של הדפדפן.

📥 אזור הקלט (Input Area)
נקודת הכניסה של הנתונים החשודים שמצאת במהלך המבדק.

רב-גוניות: ניתן להזין טקסט חופשי, להדביק קוד, או לגרור קבצים (כמו קבצי EXE חשודים או תמונות המכילות סטגנוגרפיה).

עבודה במקביל: ניתן לפתוח לשוניות (Tabs) שונות לקלטים שונים, מה שמאפשר להשוות בין תוצאות של מספר Payloads בו-זמנית.

📤 אזור הפלט (Output Area)
כאן מתקבלת התוצאה הסופית – ה-Flag, הקוד המפוענח או ה-Payload הנקי.

Replace input with output: כלי חזק מאוד ב-PT. אם הגעת לשלב ביניים שבו הפלט נראה כמו קידוד אחר, לחץ על הכפתור הזה כדי להפוך את הפלט לקלט חדש ולהמשיך "לחפור" פנימה.

Save to file: שימושי מאוד כשמפענחים קבצים בינאריים או Malware ורוצים לשמור את התוצאה כקובץ להמשך ניתוח במכונה וירטואלית.

מתודולוגיית העבודה ב-CyberChef (התהליך המחשבתי)
לפני שרצים ללחוץ על כפתורים, עובדים לפי סדר פעולות קבוע (SOP):

קביעת מטרה (Objective): הגדרת היעד. מה אני מנסה להשיג? (למשל: "יש לי מחרוזת ג'יבריש שמצאתי ב-Cookie, ואני רוצה לדעת אם יש בה שם משתמש").

הזנת נתונים (Input): העלאת הנתונים החשודים לאזור הקלט.

בחירת פעולות (Operations): בחירת הכלים המתאימים. אם הנתונים נראים כמו הצפנה, נחפש בקטגוריית Encryption/Encoding פעולות כמו Base64, ROT13 או Hex.

בדיקת תוצאה (Output): האם השגנו את המטרה? אם קיבלנו טקסט קריא – ניצחנו. אם עדיין יש ג'יבריש – חוזרים לשלב 3 ומנסים כלים אחרים.




🛠️ קטגוריות כלים קריטיות (Cheat Sheet)
1. מחלצים (Extractors) - "מציאת המחט בערימה"
בתור PT, לעיתים קרובות תקבל קובץ לוג ענק או פלט של סריקה. במקום לחפש ידנית:

Extract IP addresses: מוציא את כל כתובות ה-IP (v4 ו-v6).

Extract URLs: מוציא כתובות אתרים (חובה שיופיע פרוטוקול כמו http).

Extract email addresses: מוציא כתובות מייל לפי תבנית ה-@.


2. פורמטי נתונים (Data Format) - "פיצוח השפה"
הקידודים הנפוצים ביותר שתפגוש:

Base64: הסטנדרט להעברת נתונים בינאריים כטקסט. זיהוי מהיר: אותיות גדולות, קטנות, מספרים, ולפעמים = בסוף.

Base58: דומה ל-Base64 אבל בלי תווים מבלבלים כמו 0 (אפס) ו-O (האות O). נפוץ מאוד בעולם הבלוקצ'יין.

URL Decode: הופך תווים כמו %3A חזרה ל-:.

3. זמן ותאריך (Date / Time)
UNIX Timestamp: ייצוג זמן בשניות מאז ה-1/1/1970. אם אתה רואה מספר ארוך כמו 1725654622, זה כנראה זמן שצריך להפוך לטקסט קריא בעזרת From UNIX Timestamp.
